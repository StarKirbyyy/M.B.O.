## 选题一：“随机应变”的城市深度游助手 (City Flâneur Agent)

**核心痛点：** 传统的旅游攻略是静态的，但现实中会遇到：天气突变、景点临时闭馆、体能消耗过快或突然对某个领域（如：古建筑）产生浓厚兴趣。

- **非确定性场景：**
  - 用户初始目标很模糊：“我想在上海度过一个有艺术感的下午，但不想太累。”
  - 执行中遇到突发：Agent 计划去美术馆，但通过 API 发现今日闭馆或排队过长，需立即触发 **Self-correction**。
- **建议外部工具：**
  1. **环境感知工具：** 调用天气 API 或高德/Google Maps 实时路况与地点详情。
  2. **个性化偏好库：** 一个可以动态读写用户历史喜好的记忆组件（Long-term Memory）。
- **可视化重点：** 展示 Agent 如何从“寻找目的地”切换到“因闭馆而重新规划”的思维链路（Thought -> Action -> Observation）。



## 迭代二任务介绍 – 任务一

目标：搭建一个面向特定领域的，具备自主思考和行动能力的LLM Agent应用  

主题：不限，生活类、科研助手类、代码开发类、软工任务类…

基本需求点：

1. Agent需展现出规划和思考能力，而不是简单预定义的workflow；

2. Agent具备与外部世界交互的能力，根据自选主题，至少设计实现2个主题相关的外部工具；

3. Agent需具备自我修正能力，能够捕获系统中的异常并处理；

4. Agent需可视化内部状态流转情况（cli，web等各种形式均可）

加分项：

• Agent有机结合不止一种推理框架，如ReAct，plan-and-execute，reflexion等
